Introduction

This document contains the release notes for Fez version 1.2.0 Beta. The following sections describe the release in detail and provide late-breaking or other information that supplements the main documentation.

This is an early access beta release for wide evaluation. Your feedback is important to us, please help us make this the best Fedora web mangement interface possible. Keep in mind that we are continuing to work on Fez and things may change in the future.

See the documentation for a complete description.

Please use our Sourceforge site for submitting bugs, forum, feature requests and support.

Installation and Notes

Installation
See the installation instructions for full details.
New Features
  • Security able to be set down to the collection, object and datastream level.
  • Batch imports now done in the background and monitored through a progress bar style tracker
  • Added 'Author ID' area so authors can be uniquely identified, even if their names are displayed differently in objects.
Improvements/Bugfixes
  • Rewrote much of the SQL to improve listing performance and be compatible with MySQL 5
  • Fixed recently added list to show Today, Yesterday and last thursday if thursday is todays day name
  • Fixed a javascript bug in IE If there are no images to preview then don't try to run the preview function.
  • Updated nusoap to the latest version 1.95
  • Added thumbnail preview for 'My Fez' records
  • Fixed problem where controlled vocab selector was only saving the first of the multiple selections.
  • Added latest version of PEAR Benchmark/Timer.php
  • Changed 'Done' to 'Save Changes' in the default workflow buttons
  • Added default value for checkboxes in the XSDMF
  • Added extra install notes for retain pids and lib-tidy
  • Removed any reference to xsdmf_fez_title
  • Removed xsdmf_fez_title from sql database
Remaining Fez Issues/Bugs
  • None known.

Known Problems and Workarounds

  • Files attached to objects show their file size as 0 no matter what size they are. This is a bug in Fedora 2.1b (and earlier) and has been reported in the Fedora Bugzilla here: http://phad.lib.virginia.edu/bugs/show_bug.cgi?id=122
